House Model (Mongoose)

models/House.js

Struktur für Immobilie, letzte Wartungen und automatisch berechnete nächste Termine.

{
  owner_name: String,
  address: String,
  build_year: Number,
  heating_type: String,
  heating_install_year: Number,
  last_heating_service: Date,
  roof_install_year: Number,
  last_roof_check: Date,
  window_install_year: Number,
  last_smoke_check: Date,
  next: {
    heating: Date,
    roof: Date,
    windows: Date,
    smoke_check: Date
  }
}

REST-Endpunkte

api/index.js
  • POST /houses – neues Haus anlegen, nächste Termine berechnen
  • GET /houses – alle Häuser des Nutzers
  • GET /houses/:id – Details inkl. Historie & Next
  • PUT /houses/:id – Wartung aktualisieren, Next neu berechnen

Intervall-Logik: Heizung 2 Jahre, Dach 4 Jahre, Fenster 15 Jahre, Rauchmelder jährlich.

Push Notifications

optional

Expo Notifications oder Firebase Cloud Messaging binden Geräte-Token an Nutzer und senden Erinnerungen, sobald ein Termin < 30 Tage entfernt ist.

Für Prototyp: Backend loggt Reminder und das Frontend zeigt Badges.

Architektur

Tech-Stack & Deliverables

Frontend

React Native + Expo

Screens: Login/Signup, Dashboard, Haus-Detail, Haus-Formular, Wartungs-Timeline. Navigation via React Navigation.

State Management: React Context oder Zustand. Offline-first möglich mit SQLite als Alternative zu MongoDB.

Backend

Node.js + Express

api/index.js startet den Server, verbindet MongoDB via Mongoose und bietet CRUD-Endpunkte für Houses.

Service-Funktion berechnet next-Objekt beim Erstellen/Updaten.

Datenbank

MongoDB

Production: MongoDB Atlas; Dev: lokale Instanz oder SQLite/In-Memory-Fallback.

Collections: houses (owner, specs, last services, next), users (JWT für Auth).
